Link-Cut Tree

Posted on 2017-11-19 | In 学习笔记 | Comments: | Views:

树上的问题，我们往往可以将其转化为区间问题进行求解，对于以前的静态树，树链剖分可谓是无所不能，同时也造就了区间问题扔树上的毒瘤出题人，然而，对于一些会动的树，由于树链剖分中用到的线段树并不能支持插入和删除操作，所以，树链剖分在此便无能为力了。

这时，Tarjan 又出现了（OrzTarjan爷），结合了更加灵活的splay，Link-Cut Tree就这么诞生了。

基础知识

  • splay
  • 树链剖分
  • Top Tree

一些概念

不是道上面的东西也不要紧 (Top Tree一定要会)，先看一下基础的理论知识

  • Link-Cut-Tree (简称LCT)是解决动态树类问题一种数据结构
  • Preferred Child：重儿子，重儿子与父亲节点在同一棵Splay中，一个节点最多只能有一个重儿子
  • Preferred Edge：重边，连接父亲节点和重儿子的边
  • Preferred Path ：重链，由重边及重边连接的节点构成的链
  • Auxiliary Tree ：由一条重链上的所有节点所构成的Splay称作这条链的辅助树

浅谈辅助树

来一波图片形象体会一下


图一为原树，图三为原树的辅助树，图二为对节点 l 进行access操作后原树的形态结构（access操作就是把当前结点到根节点的路径变成重路径，相当于，把一条链放到了一棵splay中然后就可以进行各种高难度操作了，后面会有详细讲解），其中，重链已加粗，特别的，只有一个节点构成的splay用了加粗的圆圈标注。

在辅助树中，每一颗splay维护了一条重链，对于原树中的链与链之间的连接，在树链剖分中，我们把链的顶端，也就是一条链中深度最小的那个节点，与父节点的连边，而在动态树的辅助树中，我们把维护当前链的那棵splay的根节点直接与另一条链上相连的那个节点相连（因为是一棵树，所以链与链之间一定只有一处相连），并不是链上深度最小的那个链的顶端，这一点要尤其注意。

基础操作与函数

  • who() //用于判断当前节点是父节点的哪一个儿子
  • is_root() //用于判断当前节点是不是splay的根节点，注意不是原树的根节点
  • maintain() //一些更新操作，从子节点更新当前节点的信息，这里以size为例
  • reverse() //区间翻转，由于splay中是以深度为键值来维护的，进行换根操作时所有深度取反，在splay中就相当于交换左右儿子
  • pushdown() //标记下传，以翻转标记为例，将原树的根节点到当前节点路径上的标记全部下传
  • rotate() & splay() //splay的基本操作之一，但注意细节的区分，如is_root的判断
  • access() //各种高难度操作的基础，使该节点至原树的根节点的路径上的所有点构成一颗splay，即修改该路径为重链
  • make_root() //组合技，换根，对于换根操作，树链剖分也是可以搞的，但只是记录当前根是什么，然后在查询和修改时容斥一下，算不上真正的换根
  • root() //查询当前节点所在原树的根，可当高级并查集食用
  • link(x, y) & cut(x, y) //动态树必杀技，函数功能看名字就好
