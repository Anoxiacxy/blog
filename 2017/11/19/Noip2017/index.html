小凯的疑惑

题意：给出两个互素的数 a , b，求不满足 c = ax + by 的最大的 c ，其中 x , y 为非负数

题解：考虑到 a , b 互素，那么对于所有 a , a × 2 , a × 3 … a × (b - 1)，都与 b 互素，并且这些数对于 b 的模都各不相同且不为零， 将上式 c = ax + by 移项，得到 y = (c - ax) / b，对于任意给定的 c，如果其有解，那么一定可以找到一个 ax，使得 ax 与 c 在模 b 意义下同余，如果其无解，那么一定是在 0 ~ c 之间，找不到一个 ax 与 c 同余，考虑这个数 a × (b - 1)， 不存在小于这个数的 ax 与这个数同余，但其自己可以和自己同余，且 y 值刚好等于 0 ，所以我们把这个数减去一个 b，即 a × (b - 1) - b = a × b - a - b，显然，这就是最大的一个无法被构造出来的那个 c。

宝藏

题意：有一些藏宝屋需要被连通，初始选定一个藏宝屋，之后向四周的藏宝屋挖地道，每个地道的价格为 L * K，其中 K 为初始花费，L 为当前藏宝屋距离初始藏宝屋的距离，求出打通所有藏宝屋的最下花费。

题解：考场上看到数据范围为 12 就想到状压 DP 了，但发现如果状压只保存当前已选的点集是不够的，因为边权和距离有关，更像是要保存边集，但显然本质不同的边多达 72 条，不可能状压，一时也没有更好的办法就打了个搜索，直接爆搜选择哪些边，因为我们最多选择 n - 1 条边，再加上神奇的剪枝，复杂度还是可以凑合的，果断码 A*，估价函数就可以用不考虑距离的未选择点集的最小生成树，预处理一下复杂度也不高，就这么打了下来，然而之后才发现，未选择点集的最小生成树并不一定最优，可能未选择点集之间连边的花费远大于与已选点集的连边，这样我就会被卡掉了，但显然，我那个最小生成树没考虑距离，在随机数据下或者边权相同时都是可以跑过的，当然这是后话，ccf也算良心，这个错误的瞎剪枝竟然还混了 65 分，话说不打这些瞎剪枝还稳稳地有 70 分。然后现在说下正解，对于已选的点集这个状态，我们再开一维，记已选点集的最大深度，那么之后所有转移的点，都可以视为从这个最大的深度转移过去，这样做是没有问题的，因为假如新加入的点的深度不到当前点集的最大深度，那么一定会有某个之前已经算过的状态，将这个点的深度设为了一个较小值，有点玄乎吧，语言表达较差，大家意会一下

然后上代码，d 表示一个点距离一个点集的最小花费， g 表示从一个点集转移到另一个点集的最小花费，以上的花费都是在不考虑离最初选的那个点的距离时的花费，最后 f 数组表示当前最大距离为 c 时，已选点集为 now 时的最小花费，方程就不解释了，比较简单。
